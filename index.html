🛠️ OPTION B: Advanced Setup

Control security keys and DB credentials before launch

Power Users

Extract the downloaded ZIP server bundle, open the following configuration files in a plain text editor on your host computer (like Notepad, TextEdit, or VS Code), and update the values:

docker-compose.yml YAML Config
# 1. Change POSTGRES_PASSWORD to a strong database password:
POSTGRES_PASSWORD: "super_secret_password_123"
# 2. Update PGRST_DB_URI with your same password:
PGRST_DB_URI: "postgresql://authenticator:super_secret_password_123@db:5432/postgres"
# 3. Change PGRST_JWT_SECRET (At least 32 characters long):
PGRST_JWT_SECRET: "my_very_secure_jwt_secret_key_123456789"
init-db.sql SQL Script
-- Ensure the JWT secret matches the docker-compose secret exactly:
DECLARE secret text := 'my_very_secure_jwt_secret_key_123456789';

Open your system's Command Terminal (Terminal on Mac/Linux, PowerShell or system CMD on Windows OS host), use cd to navigate to the server folder containing the extracted files, and execute:

docker-compose up -d

Wait about 15-20 seconds for database containers to boot fully and initialize schemas.

Query your host computer network settings to map target server IP address:

MICROSOFT WINDOWS (CMD / POWERSHELL)

Open Command Prompt, enter ipconfig, and identify the active IPv4 Address.

APPLE MAC OSX (SYSTEM WIFI)

Open System Settings > Network > Wi-Fi > Details to copy local IP Address.

Open the GridStorage companion app, browse to Server Configuration inside side menu, write configuration URL exactly as shown (e.g. http://YOUR_IP_ADDRESS:3000), and tap Save & Verify Connection.

🛑 Troubleshooting & FAQ

"Server is Unreachable / Timeout"

Ensure both your smartphone host client and target hosting machine system are connected physically directly to the exact same local Wi-Fi router network. If running Windows hosts, configure local inbound firewall parameters to allow raw traffic through TCP Port 3000.

How do I wipe or reset database storage nodes?

Fire up shell nodes within project workspaces, and execute container cleanup commands to purge volumes:

docker-compose down -v Deletes Data

Warning: Purging physical volume allocations instantly and permanently erases all active inventories, schemas, and configurations! Initiate docker-compose commands again afterwards to mount blank node tables.
